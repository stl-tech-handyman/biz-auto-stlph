# Swagger UI для тестирования API

## Быстрый старт

### Вариант 1: Через сервер (рекомендуется)

1. **Убедитесь, что сервер запущен:**
   ```bash
   cd go
   .\start-local.ps1
   ```

2. **Откройте в браузере:**
   ```
   http://localhost:8080/swagger
   ```
   или
   ```
   http://localhost:8080/swagger-ui
   ```

3. **Установите API Key:**
   - В правом верхнем углу введите: `test-api-key-12345`
   - Нажмите "Set & Reload"

4. **Тестируйте endpoints:**
   - Найдите нужный endpoint (например, `/api/stripe/final-invoice`)
   - Нажмите "Try it out"
   - Заполните параметры
   - Нажмите "Execute"

### Вариант 2: Статический файл (если сервер не работает)

1. **Откройте файл напрямую:**
   ```
   go/swagger-ui-simple.html
   ```
   (Двойной клик в проводнике)

2. **Установите API Key** в правом верхнем углу

3. **Тестируйте!**

## Доступные endpoints

- `/swagger` - Swagger UI (через сервер)
- `/swagger-ui` - Альтернативный путь
- `/api/openapi.json` - OpenAPI спецификация в JSON
- `/api/openapi.yaml` - OpenAPI спецификация в YAML

## Если не работает

1. **Проверьте, что сервер запущен:**
   ```bash
   curl http://localhost:8080/api/health
   ```

2. **Перезапустите сервер** после изменений в коде

3. **Используйте статический файл** `swagger-ui-simple.html` как запасной вариант
